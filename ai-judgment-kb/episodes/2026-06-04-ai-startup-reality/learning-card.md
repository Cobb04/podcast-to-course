# Learning Card: AI 创业真实复盘 — 从垂直 Agent 幻觉到 toC 觉醒

**Episode:** 十字路口 × 梦奇（in vocal 创始人） | 2026.06.04

## Core Question
当所有人都在说"垂直 Agent 是 AI 创业的未来"，一个真正做了 6 个月垂直 Agent 的创始人为什么得出完全相反的结论？

## 3 Core Judgments

**1. 垂直 Agent 是结构性困局，不是执行问题。**
🟡 Medium | 梦奇 6 个月实战 + 旧金山调研多家垂直 Agent 公司
→ 服务专业用户 → 产品必然 SARS 化；服务非专业用户 → 公司必然 Agency 化。两条路终点都不是"垂直 Agent 产品"。

**2. Coding AI 越强，做好软件反而越难。**
🟡 Medium | 梦奇的客户端产品 3 个月迭代 49 个版本才到"能用"
→ MVP 成本趋近于零，但 15,000 个细节一个都不会少。真正稀缺的不是 idea，是吃细节的耐心。

**3. AI PM 的核心能力是"减步骤"而非"加功能"。**
🟡 Medium | 梦奇 Click 产品的成功验证
→ 产品价值 = 缩短用户意图到解决方案的距离。每减一步，价值翻倍。

## 3 Counter-Intuitive Insights
- **"垂直 Agent"听起来性感，落地必然不性感。** 你不可能同时服务专业用户和非专业用户——选一边，接受那个终局。
- **应用层创业公司不要招算法研究员。** 模型厂商随便改一改比你训练几个月收益大得多。招算法是给投资人看的。
- **Reddit 是最好的 AI 产品用研工具。** 不发产品帖（流量不好），纯粹讨论需求和痛点。每个决策发 40-50 帖验证。

## Framework: Gym vs Whale + 意图缩短

**Gym vs Whale：** AI toC 只有两种商业模式。Gym = 订阅制，赌用户不完全消耗 credit；Whale = 少数高 ARPU 用户撑起收入。选一个，别走中间。
**意图缩短：** 产品价值 = 用户完成任务所需步骤的倒数。PM 的工作是让步骤变少，不是让功能变多。

## 5 Action Suggestions
1. 用"垂直 Agent 死亡螺旋"审视你的产品——客户是专业还是非专业？画出终局
2. 做一次"牵引力审计"——上周的决策主要被谁驱动？（同行/投资人/员工/用户）
3. 画出核心用户任务的步骤链，找到一个可以变成按钮的步骤
4. 数一数过去一个月发了多少版本——< 5 的话，你们还在 MVP 阶段
5. 去 Reddit 发一个需求验证帖，不提产品只讨论需求

## Self-Test
你的 CTO 说要做垂直 Agent。问两个问题就能判断他是不是认真的：(1) 客户是专业用户还是非专业用户？(2) 你能接受那个终局吗？
