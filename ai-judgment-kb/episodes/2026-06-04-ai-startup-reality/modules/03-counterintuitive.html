Module 3

反直觉发现：垂直 Agent 为什么"逻辑都对，现实全错"

这期最硬核的部分——梦奇用 6 个月的亲身经历证明了一个反直觉结论：垂直 Agent 是一个结构性困局，不是执行问题。

🔑 核心洞察你不是"没把垂直Agent做好"，而是这个品类本身就不成立。服务专业用户→SARS化，服务非专业用户→Agency化。两个终点都不是"产品"。

垂直 Agent 死亡螺旋

梦奇
"大部分垂类 agent 公司都面临一个结构性的困局。你的客户其实不用你的产品，但你又想挣钱，所以就会变成你出一个人去服务他。垂直 agent 的公司在那个过渡阶段都要被迫成为一个 agency。"
🔑 可操作洞见

两条死路：
路 A：服务专业用户 → 他们会一直提需求直到你的产品长得像 Adobe → 你变成了 SaaS
路 B：服务非专业用户 → 他们不想用产品只想拿结果 → 你变成 agency 亲自交付
两条路的终点都不是"垂直 Agent 产品"。

何时使用：下次有人说"我们做 XX 领域的垂直 Agent"，先问"你的客户是专业用户还是非专业用户？"——从答案就能预测产品的终局。

反直觉 #2：Coding AI 越强，做好软件越难

梦奇
"在 coding agent 越来越强的情况下，反而我对软件这个事情更充满敬畏之心。它就是由 15000 个细节组成的。你多吃一个，用户体验好一点；少吃一个，就差一点。听完别人说一周能做出我的产品，我现在对这种言论都嗤之以鼻。"
🔑 可操作洞见

Coding AI 消灭了"从 0 到 1"的门槛，但没有消灭"从 1 到 100"的细节。做了 49 个版本才到"能用"的状态——真正稀缺的不是 idea 或 MVP，是愿意吃 15,000 个细节的团队。

何时使用：评估竞品时不要看 demo，看他们的版本迭代频率和 changelog 细节密度。49 个版本 vs 3 个版本，差距不在技术，在耐心。

反直觉 #3：应用层创业公司不要招算法研究员

梦奇
"大部分的纯应用的创业公司，不要找任何一个算法。你用不上。你快放人家去训模型。你让人家给你在那干什么？就神经病。纯属给投资人看——'我加这么一人，我多融点钱'，其实完全没有用。"
🔑 可操作洞见

应用层公司的竞争力在场景理解 + 交互设计 + 细节打磨，不在模型训练。一个应用公司招算法研究员，大概率是融资叙事需求，不是业务需求。模型厂商随便改一改，比你训练几个月带来的收益大得多。

何时使用：面试时问创始人"你们的算法团队在做什么？"如果回答含糊——信号。

场景判断：这个垂直 Agent 能活吗？

一家创业公司做"法律合同审查 Agent"。创始人是律师出身，产品能自动找出合同中的风险条款。客户有两种：大所合伙人（专业用户）和中小企业主（非专业用户）。目前两类客户都在提需求。用梦奇的框架，这家公司最可能走向哪个终局？

这正是梦奇的框架最直接的预测。大所合伙人会要求"这里我需要手动标注""那个条款的判断标准要调整"→产品逐渐变成法律 SaaS。中小企业主会说"你别让我看界面了，直接帮我把合同审了"→你得亲自交付。两个方向都能挣钱，但都不是"垂直 Agent 产品"这个叙事。
