# Episode Knowledge Entry: AI 创业真实复盘

**Episode:** 十字路口 × 梦奇（in vocal 创始人） | 2026.06.04

## Core Thesis
垂直 Agent 是结构性困局——服务专业用户导致产品 SARS 化，服务非专业用户导致公司 Agency 化。AI 创业的真正壁垒不是技术或 idea，是吃 15,000 个细节的耐心 + 让用户牵引（而非虚荣牵引）产品决策的定力。

## Reusable Frameworks
- **垂直 Agent 死亡螺旋**: 专业用户 → SaaS 化 / 非专业用户 → Agency 化。用于判断垂直 AI 产品的终局。
  - **Relationship to existing KB**: NEW — 与 Harness 三层架构不同维度，互补（Harness 讲技术架构，死亡螺旋讲商业模式）
- **Gym vs Whale 模型**: AI toC 两种商业模式。Gym = 订阅制赌不使用，Whale = 少数高 ARPU 用户。
  - **Relationship to existing KB**: NEW
- **意图缩短度量**: 产品价值 = 1 / (意图→解决方案的步骤数)。PM 的核心工作是减步骤。
  - **Relationship to existing KB**: NEW — 与"More Context, Less Control"形成呼应（都强调"少即是多"）
- **15000 细节原则**: 软件质量 = Σ(被吃掉的细节)。Coding AI 降低 MVP 成本，但不减少细节总量。
  - **Relationship to existing KB**: EXTENDS [[Harness 三层架构]] — harness 的第二层（上下文管理）就是 15,000 细节的一部分
- **Reddit 决策漏斗**: 用研方法——数据挖掘 → 发帖验证 → 回收迭代。每个决策 40-50 帖。
  - **Relationship to existing KB**: NEW
