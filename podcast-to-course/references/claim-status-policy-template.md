# Claim Status Policy

Purpose: prevent podcast claims from entering the wiki as unquestioned truth.

## Statuses

| Status | Meaning | Wiki Behavior |
|---|---|---|
| accepted | Strong enough to use as a working judgment. | Can appear in framework pages and course as a usable rule. |
| tentative | Plausible but not fully validated. | Can appear, but must carry caveats and evidence needs. |
| challenged | AI or external sources found a serious weakness. | Must appear with Debate Arena link before reuse. |
| disputed | Credible sources conflict. | Do not use as a rule without showing both sides. |
| outdated | Was likely true before but may no longer be current. | Requires web check before reuse. |
| rejected | Unsupported or contradicted enough to remove. | Keep only in archive/learning notes, not active wiki knowledge. |

## Rule

No high-impact claim can enter a framework as `accepted` unless it has:

1. transcript anchor;
2. evidence level;
3. status;
4. update condition;
5. Debate Arena review if the claim affects product, business, or technical decisions.

## When To Use Web Search

Use web search for:

- product status;
- company metrics;
- market outcome;
- model capability;
- legal/regulatory facts;
- research claims;
- any claim likely to change after the recording date.

Do not rely on web search alone for:

- value judgment;
- strategic taste;
- user psychology;
- causal interpretation of a private company project.
